详细描述

封装需要Login hash confirm 的通信

服务器端的部分通信需要确认身份。
确认身份的方式是通过Login hash confirm 。

通信协议

首先，这个通信协议只定义了基本通信协议 中data的部分。
查看完整的通信协议，请见基本通信协议 。
这是一个有状态的连接。也就是说，在通过Login hash confirm 之前和之后的通信协议并不相同。

  • 之前 ：
    • 客户端向服务器端 : <int32:userId> <uint32:size> <byte[size]:code>
      1. 32位有符号整数，表示客户端用户ID。
      2. 32位无符号整数，表示Login hash code 的长度。
      3. size个字节，表示Login hash code 。
    • 服务器端向客户端 ： bool
      1. 表示Login hash confirm 的结果。
  • 之后 ： 由子类定义。
    参见:
    afterLhc()
